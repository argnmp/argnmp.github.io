cloud native computing: approach in software development that utilizes cloud computing to build and run scalable applications

container: isolated process running in the existing host OS

microservices: small, logic-focused services that deal with one responsibility

microservices 간의 sync 통신 방식은 RESTApi, async 통신 방식은 message queue

scale out은 instance의 개수를 늘리는 것, scale up은 microservice에 할당된 resource를 늘리는 것

devops는 개발과 운영을 밀접하게 연계해서 수행하는 것을 의미

ci/cd는 application development 단계에서 자동화를 도입해서 사용자에게 자주 배포하는 방식을 의미

kubernetes는 open-source container orchestration system이다. 여러 호스트로 구성된 cluster 간의 application container의 development, scaling, operation을 자동화하는 플랫폼

kubernetes는 platform for building platforms

kubernetes는 distributed application과 cluster 간의 interface, application 개발자는 core business logic에만 집중하면 된다.

container의 장점

  • isolated environments
  • much lighter than VMs
  • Faster start-up time

docker: platform for packaging, distributing, running applications

docker는 application과 application이 동작하는 환경을 같이 packaging 한다.

container image는 여러 layer로 구성되어있고 공유되어 사용된다. 없는 layer만 다운로드 하면 되고, CoW 방식으로 메모리에서 공유된다.

docker run -it ubuntu bash: interactive tty 모드로 동작하겠다. ubuntu image에서 bash 프로그램을 실행

docker compose는 여러 microservices를 build, run, manage 하는 편리한 환경을 제공한다.

linux에서 container 제공이 가능한 이유는 linux namespace, cgroups, capabilities 덕분
