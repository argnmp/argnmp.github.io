문제

풀이

#include <iostream>
#include <algorithm>
#include <climits>
#include <vector>
#include <queue>
#include <map>
#include <unordered_map>
#include <set>
#include <unordered_set>
#include <bitset>
#include <cstring>
#include <cmath>
using namespace std;
using ll = long long;
int n, m, k;
ll value[1000000] = {0,};
ll segt[2097152] = {0,};

void segt_init(int l, int r, int node){
    if(l==r){
        segt[node] = value[l];
        return;
    }
    segt_init(l, (l+r)/2, node*2);
    segt_init((l+r)/2+1, r, node*2+1);
    segt[node] = segt[node*2] + segt[node*2+1];
}
void segt_replace(int l, int r, int node, int t, ll v){
    segt[node] -= value[t];
    segt[node] += v;
    if(l==r){
        value[t] = v;
        return;
    }
    int mid = (l+r)/2;
    if(t <= mid){
        segt_replace(l, mid, node*2, t, v);
    } else {
        segt_replace(mid+1, r, node*2+1, t, v);
    }
}
ll calc(int l, int r, int node, int lt, int rt){
    // printf("%d, %d, %d, %d, %d\n", l, r, node, lt, rt);
    if(l==lt && r==rt)
        return segt[node];
    int mid = (l+r)/2; 
    if(rt <= mid)
        return calc(l, mid, node*2, lt, rt);
    else if(mid < lt)
        return calc(mid+1, r, node*2+1, lt, rt);
    else
        return calc(l, mid, node*2, lt, mid) + calc(mid+1, r, node*2+1, mid+1, rt);
}
void solve(){
    cin >> n >> m >> k;
    for(int i = 0; i<n; i++)
        cin >> value[i];
    segt_init(0, n-1, 1);
    ll a, b, c;
    for(int i = 0; i<m+k; i++){
        cin >> a >> b >> c;
        if(a==1){
            segt_replace(0, n-1, 1, b-1, c); 
        }
        else if(a==2){
            cout << calc(0, n-1, 1, b-1, c-1) << '\n';
        }
    }
}
int main(){
    ios::sync_with_stdio(0);
    cin.tie(0);
    solve(); 
    return 0;
}

n개의 수가 있을 때 구간의 합을 구하기 위한 방법으로, 1~k 구간의 합을 미리 저장해두어서 한번의 뺄셈 연산을 사용할 수 있다. 문제에서 주어진 수의 개수의 최댓값은 1000000이다. 계산에 필요한 값을 미리 저장해두는데 필요한 시간은 O(n)이고 구간의 합을 구하는 데 필요한 시간은 O(1) 이므로, 주어진 시간 내에 충분히 해결이 가능하다.

하지만 문제는 값이 갱신될 수 있다는 것이다. 위의 방식으로 구간의 합을 구하는 경우 시간 내 해결이 불가능하다. 첫번째 수를 갱신한다면 미리 계산해둔 구간의 합들을 모두 수정해야 하기 때문에, 최악의 경우 n * m = 1000000 * 10000 의 연산이 추가로 필요하기 때문이다.

segment tree를 활용하면 O(lgN) 의 시간에 구간 합을 구할 수 있다.

출처: https://www.geeksforgeeks.org/segment-tree-data-structure/?ref=gcse

segment tree는 segment의 정보를 담고 있는 tree 형태의 자료구조이다. 구간 합 문제를 해결하기 위해서는 binary tree를 이용한다. 부모 노드는 i ~ k 구간의 구간 합을 저장하고, 두 자식 노드는 각각 이를 반으로 나눈 i ~ (i+k)/2 와 (i+k)/2 + 1 ~ k 구간 합을 저장한다.

루트 노드에는 전체 구간의 합을 저장하고, 자식 노드로 내려갈 때마다 각각 반으로 나눈 구간의 합을 저장한다. 트리 정보의 저장은 배열을 이용한다. 루트 노드의 index를 1이라고 설정 했을 때, 부모 노드의 index가 k 이면 자식 노드의 index는 각각 k*2 와 k*2+1 로 표현이 가능하다.

이러한 표현에서, segment tree는 full binary tree 또는 perfect binary tree가 된다. 전체 수의 개수를 N이라고 했을 때, N이 2의 거듭제곱 형태인 경우가 perfect binary tree이다.

N이 8 = 2^3 일 때, perfect binary tree 이고 lg8 = 3 이므로 트리의 높이는 3이며 2^0 + 2^1 + 2^2 + 2^3 = 2^4 - 1 = 15 개의 노드를 필요로 한다. N이 4= 2^2 일 때, perfect binary tree 이고, 트리의 높이는 2이며, 2^3 - 1 = 7 개의 노드를 필요로 한다.

N이 7인 경우에는, N이 4일 때 보다 노드의 개수가 많고 N이 8일 때 보다 노드의 개수가 적다. 따라서 N이 8일 때 필요한 노드의 개수 만큼, tree를 저장하는 배열의 공간을 미리 할당해두면 된다.

즉, 2^(ceil(lgN)+1) - 1 의 tree를 저장하는 배열이 필요하다.

문제에서는 N의 최댓값이 1000000 이다. 따라서 2^21 - 1 정도의 공간을 미리 할당해두면 된다.

segment tree의 각 노드의 값을 계산하는 것은, leaf 노드 부터 재귀적으로 호출하며 올라오면 된다. 위의 코드에서 segt_init에 해당한다. 대략 노드 개수 만큼의 함수 호출이 필요하다.

구간합의 계산은 다음과 같은 방식으로 수행한다.

  • 현재 노드에 해당하는 구간의 mid index(왼쪽 자식노드와 오른쪽 자식노드를 구분하는 index)을 구한다. 위의 코드에서 segment tree 생성 시, 왼쪽 자식 노드에 mid index를 포함하도록 하였다.
  • 다음의 세가지 경우의 수가 존재한다.
    • 구해야 하는 구간의 가장 오른쪽 index가 mid index보다 작거나 같으면, 구해야 하는 구간을 왼쪽 자식 노드로 재귀 호출한다.
    • 구해야 하는 구간의 가장 왼쪽 index가 mid index + 1 보다 크거나 같으면, 구해야 하는 구간을 오른쪽 자식 노드로 재귀 호출한다.
    • 이외의 경우의 수는 왼쪽 자식 노드와 오른쪽 자식 노드 모두에 걸쳐있는 경우로, 왼쪽 오른쪽 자식 노두 모두로 필요한 구간을 재귀 호출한다. 위의 코드에서 calc에 해당한다.

갱신은 다음과 같은 방식으로 수행한다.

  • 루트 노드 부터, 갱신 해야 하는 index가 포함된 자식 노드로 이동하면서 모두 업데이트 해준다.

한가지 주의할 사항이 있다. 문제에서 입력으로 주어지는 모든 수는 -2^63보다 크거나 같고, 2^63-1보다 작거나 같은 정수이다. 라고 하였다. 이는 long long int의 표현 가능한 범위에 해당한다. 주어지는 N개의 값들은, long long int 범위에서 최댓값과 최솟값이 될 수 있고, 이는 tree에 구간합을 저장해둘 때 overflow가 발생할 수 도 있음을 의미한다.

그렇다면 컴퓨터의 정수 표현형으로 전체 구간합을 표현하지 못할 수 없는 것 아닌가? 하는 의문이 들 수도 있고, 나도 그렇게 생각했다. 하지만, 단, 정답은 -2^63보다 크거나 같고, 2^63 - 1보다 작거나 같은 정수이다. 라고 하였다. 이는 문제에서 요구하는 구간합을 계산하기 위해 필요한 segment tree의 노드의 계산 값들을 더했을 때, overflow가 발생하지 않음을 의미한다. 필요한 노드들 외의 다른 노드의 구간 합들을 더했을 때 overflow가 발생하는 것은 신경 쓸 필요가 없는 것이다.

느낀점

segment tree를 알게 되었다. 또한 문제를 조금 더 면밀하게 읽어야겠다.
