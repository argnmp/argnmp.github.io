문제

https://www.acmicpc.net/problem/1019

코드

#include <cmath>
#include <iostream>
#include <cstdlib>
#include <algorithm>
#include <climits>
#include <vector>
#include <queue>
#include <map>
#include <unordered_map>
#include <set>
#include <unordered_set>
#include <bitset>
#include <cstring>
using namespace std;
using ll = long long;
string sn;
ll n;
ll co[10] = {0,};
void print(){
    for(int i = 0; i<10; i++){
        cout << co[i] << ' ';
    }
    cout << endl;
}
void n_digit_count(int size, bool exclude_high_zero){
    if(size <= 0) return;
    ll base_count = pow(10, size-1)*size;
    for(int i = 1; i<10; i++)
        co[i] += base_count;
    if(exclude_high_zero){
        int tsize = size-1;
        while(tsize>=0){
            base_count -= pow(10, tsize);
            tsize -= 1;
        }
    }
    co[0] += base_count;
}
void calc_remainder(string target, bool is_highest){
    int digit = target[0]-48; 
    int i = is_highest ? 1 : 0;
    for(;i<digit; i++){
        co[i] += pow(10, target.length()-1);
        n_digit_count(target.length()-1, false);
    }
    if(target.length() > 1){
        string modified = target.erase(0, 1);
        co[digit] += atoll(modified.c_str()) + 1;
        calc_remainder(modified, false);
    }
    else {
        co[digit] += 1;
    }
}
void solve(){
    cin >> sn;
    n = atoll(sn.c_str());
    n_digit_count(sn.length()-1, true); 
    calc_remainder(sn, true);
    print();
}
int main(){
    ios::sync_with_stdio(0);
    cin.tie(0);
    solve(); 
    return 0;
}

풀이

n이 최대 10억이기 때문에, 직접 숫자를 counting 하는 방식으로는 해결이 불가능하다. 따라서 규칙을 찾아서 해결해야 한다.

n이 543으로 주어진 경우를 생각해보자.

먼저 n의 자릿수를 살펴보아야 한다. 5432의 경우는 네 자리이다. 최대 세 자리의 수로 만들수 있는 모든 페이지는 모두 포함되기 때문에, 최대 세 자리로 만들 수 있는 모든 페이지에 속하는 숫자의 개수를 먼저 세어주어야 한다.

세 자리로 만들 수 있는 모든 수는 000~999이다. 이 수들 중에서 0의 개수는 다음과 같이 셀 수 있다.

  • 첫째 자리의 숫자가 0일 때 모든 가능한 조합(0xx): 10^2 가지 (다른 두 자리가 각각 10개의 숫자가 들어갈 수 있으므로)
  • 둘째 자리의 숫자가 0일 때 모든 가능한 조합(x0x): 10^2 가지 (다른 두 자리가 각각 10개의 숫자가 들어갈 수 있으므로)
  • 셋째 자리의 숫자가 0일 때 모든 가능한 조합(xx0): 10^2 가지 (다른 두 자리가 각각 10개의 숫자가 들어갈 수 있으므로) 따라서 총 300개의 0이 이 범위에 포함되어 있다. 뿐만 아니라, 0~9의 모든 숫자가 각각 300개씩 포함되어 있다.

하지만 책 페이지의 경우 0, 01, 02, 001 과 같은 페이지는 존재하지 않기 때문에 이러한 경우의 0의 개수를 빼주어야 한다. 그러한 경우의 수는 다음과 같다

  • 첫째 자리의 숫자가 0일 때 모든 가능한 조합(0xx): 10^2 가지
  • 첫째, 둘째 자리의 숫자가 0일 때 모든 가능한 조합(00x): 10가지
  • 첫째, 둘째, 셋째 자리의 숫자가 0일 때 모든 가능한 조합(000): 1가지 총 111개의 0이 포함되어 있다. 그래서 0의 경우는 300-111 = 189 개의 0이 존재한다.

결론적으로 최대 세 자리의 페이지에서 나타나는 각 숫자의 개수는 다음과 같다. 주어진 테스트 케이스를 통해 이 개수가 맞다는 것을 확인할 수 있다.

  • 0: 189개
  • 1~9: 각 300개

위의 함수 중 n_digit_count(int size, bool exclude_high_zero)가 이것을 계산한다. size는 자릿수를 의미하고, exclude_high_zero는 앞서 존재하지 않는 페이지에 대한 0의 개수를 빼줄지를 결정한다.

다음으로는 나머지 페이지에 대해서 숫자의 갯수를 세어야 한다.

앞서 5432의 경우 1999 페이지 까지에서 등장하는 각 숫자의 개수를 구했다. 따라서 이제는 10005432 페이지에서 등장하는 각 숫자의 개수를 구해야 한다.

앞에서 부터 한 자리씩 해당 숫자와, 이 과정에서 등장하는 다른 자리의 숫자가 등장하는 횟수를 카운팅 하면 된다. 여기서 중요한 것은 두가지를 함께 카운팅을 하는 것이다.

  1. 현재 보고있는 자리의 숫자의 총 등장 횟수
  2. 1번의 과정에서, 나타나는 하위 자리들에 있는 숫자들의 총 등장 횟수

첫번째 줄에서는 다음과 같이 카운팅해야 한다. (1xxx)

  • xxx에는 (000~999) 까지 가능하기 때문에 1은 총 1000번 등장한다.
  • xxx에는 (000~999) 까지 가능하기 때문에, 처음에 먼저 계산하는데 사용한 방식을 따라 다른 자릿수에서 나타나는 수자에 대해 카운팅 해주어야 한다 (n_digit_count 함수를 사용한다.) 이때, xxx의 맨 앞자리에 0이 나와도 가능한 경우의 수 이므로(1xxx 이기 때문) n_digit_count의 exclude_high_zero를 false로 해주어야 한다.

2xxx, 3xxx, 4xxx에 대해서도 마찬가지로 카운팅을 수행한다.

다음으로 5xxx의 경우에는 다음과 같이 카운팅해야 한다.

  • xxx에는 (000~432) 까지 가능하기 때문에 5는 총 433번 등장한다.
  • 맨 앞자릿수 5를 제거한 432는 이 과정을 똑같이 적용한다 (재귀 호출) 다만, 5432 와는 달리 맨 앞자리에 0이 등장한다는 것을 고려애햐 한다 (재귀 호출)

위의 코드에서 calc_remainder(string target, bool is_highest)함수가 이 계산을 재귀적으로 호출하며 수행한다. 카운팅할 숫자를 string 형태로 넘겨서, 앞의 자릿수를 쉽게 제거할 수 있도록 하였다. is_highest는 맨 앞자리 숫자에 0이 포함되는 지 여부를 나타내는 것으로, 위의 예시에서 5432의 경우 true로 설정되고, 그 다음부터의 재귀 호출에서는 false로 설정된다.

느낀점

풀면서 숫자들이 너무 헷갈려서 어려웠고, 또한 집중을 잘 하지 못해서 푸는 데 시간이 오래 걸렸다. 뭔가 펜으로 그림을 그려가면서 푸는 것도 좋을 듯 하다.
