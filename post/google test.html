library link 에서 라이브러리를 만들고 정적 및 동적으로 프로젝트에 연결하는 법에 대해 알아보았다. ops 라는 디렉터리를 생성해서 libops라는 라이브러리를 컴파일했엇다.

이번 글에서는 google test를 이용해서 라이브러리를 테스트하는 방법을 알아본다.

google test

https://github.com/google/googletest google test는 google의 오픈소스 c++ 테스팅 framework이다.

간단하게는 gtest라고 불린다.

먼저 https://github.com/google/googletest/releases 여기에서 tar.gz로 압축된 가장 최신의 릴리즈를 다운받는다.

먼저 압축을 해제한다.

tar -xvzf googletest-1.15.0.tar.gz

아래의 디렉터리로 이동한다.

cd googletest-1.15.0/googletest

그리고 아래와 같이 설치해준다.

cmake ..
make
make install

기본 설치 위치인 /usr/local 하위에 google test가 설치된다.

test 코드 작성하기

지난번 글에서 사용했던 ops 라이브러리에 대한 google test code를 작성하기 위해 아래와 같은 디렉터리 구조를 사용한다.

.
├── Makefile
├── include
│   └── ops.h
├── src
│   ├── ops.cpp
│   └── ops_expand.cpp
└── test
    ├── test.cpp
    └── test_expand.cpp

test 디렉터리 하위에 google test 코드를 작성한다.

Makefile도 아래와 같이 약간 변경되었다.

CXX=g++
CXXFLAGS=-std=c++20
TARGET=libops
TARGETDIR=libs
HEADERS=./include/ops.h
SRCS=./src/ops.cpp ./src/ops_expand.cpp
OBJS=$(SRCS:%.cpp=%.o)

TESTTARGET=test_main
TESTSRCS+=./test/test.cpp
TESTSRCS+=./test/test_expand.cpp
TESTOBJS=$(TESTSRCS:%.cpp=%.o)


static: $(TARGETDIR) $(OBJS)
	ar rcs $(TARGETDIR)/$(TARGET).a $(notdir $(OBJS))

dynamic: $(TARGETDIR) $(OBJS)
	$(CXX) $(CXXFLAGS) -shared -o $(TARGETDIR)/$(TARGET).so $(notdir $(OBJS))

test: $(OBJS) $(TESTOBJS) static
	$(CXX) $(CXXFLAGS) -o $(TESTTARGET) $(notdir $(TESTOBJS)) -lgtest -lgtest_main -lops -L$(TARGETDIR)
	make clean


clean:
	rm -rf *.o
	rm -rf $(TARGETDIR)

$(TARGETDIR):
	mkdir -p $@

%.o: %.cpp
	$(CXX) $(CXXFLAGS) -o $(notdir $@) -c $<

이전 글에서 처럼 정적 라이브러리로 먼저 ops 라이브러리를 컴파일한다.

그러면 /libs/libops.a 라는 결과물이 생성된다.

이후 test 디렉터리 하위의 테스트 코드들을 컴파일 한다. test.cpp와 test_expand.cpp로, 코드는 아래와 같다.

test.cpp

#include <gtest/gtest.h>
#include "../include/ops.h"

TEST(OpsTest, Assertions) {
  EXPECT_EQ(ops::add(1, 2), 3);
  EXPECT_EQ(ops::sub(1, 2), -1);
}

test_expand.cpp

#include <gtest/gtest.h>
#include "../include/ops.h"

TEST(OpsExpandTest, Assertions) {
  EXPECT_EQ(ops::mul(1, 2), 2);
  EXPECT_EQ(ops::div(4, 2), 2);
}

Makefile에서는 먼저 두개의 test 코드를 object file로 컴파일 한다.

%.o: %.cpp
	$(CXX) $(CXXFLAGS) -o $(notdir $@) -c $<

그다음 object file과 라이브러리들을 링크해준다.

test: $(OBJS) $(TESTOBJS) static
	$(CXX) $(CXXFLAGS) -o $(TESTTARGET) $(notdir $(TESTOBJS)) -lgtest -lgtest_main -lops -L$(TARGETDIR)

일단은 ops 라이브러리를 링크해주어야 하기 때문에 -lops -L$(TARGETDIR)가 필요하다.

그리고 -lgtest를 이용해서 앞서 설치했던 /usr/local 하위의 gtest 라이브러리를 링크해준다.

테스트코드를 실행하기 위한 main 함수가 필요하지만, 우리는 작성하지 않았다.

물론 main함수를 직접 작성할 수도 있지만, 작성하지 않더라도 google test에서는 libgtest_main.a을 링크함으로써 테스트 코드를 실행할 수 있다.

따라서 -lgtest_main를 추가해주었다.

아래의 명령어를 이용하면 TESTTARGET으로 지정한 test_main 을 컴파일하고 실행한다.

make test
./test_main
