linux에서 bluetooth 연결 문제 해결법

Taehyeon Kim

2023-05-13

서론

arch linux를 잘 사용하던 중 갑자기 블루투스가 되질 않았다. arch linux에 desktop manager로 gnome을 사용중이어서 보통 gnome 설정의 bluetooth 항목에서 블루투스 키보드를 연결을 한다. 잘 안되길래 컴퓨터를 재부팅 해봤는데도 먹통이었다. 이럴 경우 다음과 같은 방법으로 해결이 가능하다.

해결법

bluetooth service 확인

sudo systemctl status bluetooth

터미널을 열어서 위의 명령어를 입력한다. 그러면 정상적으로 작동하는지 확인이 가능한데, 나의 경우는 가장 마지막 줄에,

Failed to set mode: Failed (0x03)

이라는 이상한 오류가 발생했다.

rfkill block 확인

이번에는 terminal에

rfkill list

이 명령어를 입력하면 다음과 같이 뜬다.

0: hci0: Bluetooth
	Soft blocked: yes
	Hard blocked: no
1: phy0: Wireless LAN
	Soft blocked: no
	Hard blocked: no

내 컴퓨터의 경우 위의 Bluetooth 항목의 Soft blocked가 yes로 되어 있었다. block 되어 있어서 발생하는 문제였다.

rfkill block 해제

먼저 다음의 명령어로 block을 풀어준다.

sudo rfkill unblock bluetooth

rfkill list

다시 확인 해보면 yes -> no로 변한 것을 알 수 있다.

bluetooth service 재시작

그런 후 systemctl을 통해 bluetooth service를 다시 시작해준다.

sudo systemctl restart bluetooth

재연결

bluetoothctl 이나 gnome bluetooth setting으로 블루투스를 다시 연결해보면 정상적으로 작동하는 것을 확인할 수 있다.
