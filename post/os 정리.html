os 정리

Taehyeon Kim

·

2024-10-20

blog

multiprogramming 개념에서 timesharing(multithreading) 개념으로 진화했다.

timesharing은 job의 전환을 매우 빠르게 해서 동시에 동작하는 것처럼 보이도록 하는 기술

os는 interrupt driven이다. interrupt는 hardware에 의해 발생되는 것이고, exception or trap은 software에 의해 발생한다.

system call은 os가 제공하는 서비스에 대한 programming interface이다. API 형태로 사용이 가능하도록 UNIX의 POSIX와 같은 라이브러리가 존재한다.

process는 크게 text, data, heap, stack 영역으로 나뉜다.

PCB는 kernel data structure로, process(job) 간의 전환을 위해 사용된다.

Job scheduling queue를 가지고 process 들을 어떠한 순서로 실행할지 결정한다.

  • long-term scheduler는 process를 ready 상태로 만드는 역할, degree of multiprogramming을 조정한다.
  • short-term scheduler는 process를 ready 상태에서 running 상태로 만드는 역할

context switching은 overhead

process creation: 처음에는 parent와 child가 같은 address space 공유, 이후에는 새로운 프로그램을 로드

IPC

  • shared memory: producer-consumer model
  • message passing: communication link와 send(), receive() call (blocking, non-blocking 방식이 가능하다)

cpu scheduler - dispatcher : shorterm scheduler에 의해 선택이 된 process에 cpu control을 부여

  • dispatcher latency == context switch overhead

process에서 code, data, files(files + signal) 은 thread간에 공유된다. register, stack, pc 는 각 thread가 자신만의 것을 가지고 있다.

thread에는 kernel thread와 user thread가 존재

scheduling criteria

  • cpu utilization
  • throughput
  • trunaround time
  • waiting time
  • respond time

scheduling algorithms

  • FCFS: nonpreemptive, convoy effect
  • SJF: nonpreemptive
  • SRTF: preemptive (SJF에)
