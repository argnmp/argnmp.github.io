user에게 sudo 권한 부여하기

sudoers

sudoers는 linux에서 sudo 정책을 담당하는 플러그인이다.

man sudoers를 통해 명령어를 입력해보면 다음과 같은 설명이 나온다.

NAME
       sudoers - default sudo security policy plugin

DESCRIPTION
       The sudoers policy plugin determines a user's sudo privileges.  It is the default sudo policy plugin.  The policy is driven by the /private/etc/sudoers file or, optionally, in LDAP.  The
       policy format is described in detail in the SUDOERS FILE FORMAT section.  For information on storing sudoers policy information in LDAP, see sudoers.ldap(5).

/priavte/etc/sudoers 파일을 이용해서 권한을 정의한다. sudoers 파일은 /etc하위에서도 발견이 되는데, readlink를 통해 /private/etc/sudoers 에 링크 되어 있는 것을 알 수 있다. (mac os 기준)

readlink -f /etc/sudoers

sudoers파일은 visudo 라는 프로그램을 이용해서만 수정이 가능하다. 이 프로그램을 통해서 저장하기 전에 아래와 같이 syntax check를 수행한다. 만약 오류가 있다면 여러 옵션을 제공하여 다시 수정할 수 있도록 한다.

/etc/sudoers:57:12: syntax error
tyler           ALL != (ALL) NOPASSWD:ALL
           ^
What now?
Options are:
  (e)dit sudoers file again
  e(x)it without saving changes to sudoers file
  (Q)uit and save changes to sudoers file (DANGER!)

sudo 권한 추가하기

아래의 명령을 입력한다. 그러면 기본설정인 vim이 실행될 것이다.

sudo visudo

아래로 커서를 내리면 이렇게 권한이 정의되어 있는 것을 확인할 수 있다.

# root and users in group wheel can run anything on any machine as any user
root		ALL = (ALL) ALL
%admin		ALL = (ALL) ALL

root와 admin 그룹에 sudo 권한이 부여되어 있는 상태이다.

현재 내가 쓰는 계정은 tyler라는 계정이고 아래의 캡쳐 화면 처럼 admin 그룹에 들어가 있는 것으로 보인다.

그렇기 때문에 현재는 sudo권한을 사용할 수 있는 상태이다.

만약 특정 유저에 대해서 sudo권한을 부여하고 싶다면, 유저의 이름과 함께 아래와 같이 입력해주면 된다.

# root and users in group wheel can run anything on any machine as any user
root		ALL = (ALL) ALL
%admin		ALL = (ALL) ALL
tyler       ALL = (ALL) ALL

sudo 권한 사용시 비밀번호 입력 제거

# root and users in group wheel can run anything on any machine as any user
root		ALL = (ALL) ALL
%admin		ALL = (ALL) ALL
tyler       ALL = (ALL) NOPASSWD:ALL

ALL 앞에 NOPASSWD: 를 입력해주면 된다.

비밀번호 입력을 제거하는 것은 특히 나처럼 vim에서 스크립트를 이용해서 sudo권한으로 컴파일을 할 때 유용하다.

또한 여러 프로그램에서 sudo 권한이 필요한 경우, 예를 들어 sudo cargo run 과 같이 입력하지 않고 cargo run으로도 자동으로 sudo 권한이 주어진다.

cargo run 는 rust로 작성된 프로그램을 컴파일하고 실행한다. 이 과정에서 crates.io의 index를 caching 하는데, 저장되는 위치가 $HOME/.cargo이다.

그동안은 sudo 권한 사용시 sudo를 입력하여 명시적으로 sudo권한을 주고 비밀번호를 입력해야 했다. cargo run에서 컴파일은 잘 되지만, 권한 부족으로 $HOME/.cargo위치에 cache를 쓸 수가 없었다. 명시적으로 sudo cargo run을 통해서만 비밀번호를 입력하는 창이 뜨고 cache를 쓸 수가 있었다.

하지만 NOPASSWD를 추가하여, cargo run을 입력해도 자동으로 sudo권한이 주어져서 $HOME/.cargo위치에 cache를 쓸 수 있게 되었다.
