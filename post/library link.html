library를 프로젝트에 연결하기

c나 c++로 프로그래밍을 할 때 외부의 코드를 사용해야 하는 경우가 있다. 이 때 컴파일 과정에서 별도의 링크 과정을 거쳐야 한다.

외부 코드의 집합을 library라고 한다. library의 종류에는 static library와 dynamic library가 있다.

static library

빌드 결과물에 외부 코드가 그대로 포함되는 방식을 사용한다. linux에서 .a확장자로 끝나는 파일이 static library에 해당한다. ar 프로그램을 이용해서 library file을 생성할 수 있다.

dynamic library

static library와 달리 빌드 결과물에 외부 코드가 포함되지 않는다. 런타임에 동적으로 외부 라이브러리를 연결한다. linux에서 .so확장자로 끝나는 파일이 dynamic library에 해당한다.

linux에서 바이너리를 실행할 때 dynamic library를 찾아서 연결한다.

static library, dynamic library 링크 방법

library link를 테스트해보기 위해서 다음과 같은 파일들을 생성했다.

.
├── Makefile
├── main.cpp
├── ops
│   ├── Makefile
│   ├── include
│   │   └── ops.h
│   └── src
│       ├── ops.cpp
│       └── ops_expand.cpp
└── run.sh

ops library

ops 디렉터리는 외부 library에 해당한다.

ops.h

#ifndef __OPS_H__
#define __OPS_H__

namespace ops {

int add(int a, int b);
int sub(int a, int b);
int mul(int a, int b);
int div(int a, int b);

}
#endif

ops.cpp

#include "../include/ops.h"

int ops::add(int a, int b){
    return a + b;
}
int ops::sub(int a, int b){
    return a - b;
}

ops_expand.cpp

#include "../include/ops.h"

int ops::mul(int a, int b){
    return a * b;
}
int ops::div(int a, int b){
    return a / b;
}

ops.cpp와 ops_expand.cpp는 ops.h의 함수들을 구현한다.

ops library를 컴파일 하기 위한 Makefile은 아래와 같다.

ops/Makefile

CC=g++
HEADERS=./include/ops.h
SRCS=./src/ops.cpp ./src/ops_expand.cpp
OBJS=$(SRCS:%.cpp=%.o)

TARGET=libops
TARGETDIR=libs

static: $(TARGETDIR) $(OBJS)
	ar rcs $(TARGETDIR)/$(TARGET).a $(notdir $(OBJS))
dynamic: $(TARGETDIR) $(OBJS)
	gcc -shared -o $(TARGETDIR)/$(TARGET).so $(notdir $(OBJS))

clean:
	rm -rf *.o
	rm -rf $(TARGETDIR)

$(TARGETDIR):
	mkdir -p $@

%.o: %.cpp
	$(CC) -o $(notdir $@) -c $<

static library로 컴파일할 경우 ops/libs/libops.a 파일을 생성하고, dynamic library로 컴파일할 경우 ops/libs/libops.so 파일을 생성한다.

static library는 컴파일 된 object 파일들을 가지고 ar프로그램을 이용해서 생성할 수 있다.

dynamic library는 컴파일 된 object 파일들을 가지고 gcc -shared 옵션으로 생성할 수 있다.

main

위에서 빌드한 ops library로 실행 프로그램을 만들 수 있다.

main.cpp

#include <iostream>
#include "ops/include/ops.h"

using namespace std;

int main(){
    int a = 10;
    int b = 5;
    
    cout << ops::add(a,b) << '\n';
    cout << ops::sub(a,b) << '\n';
    cout << ops::mul(a,b) << '\n';
    cout << ops::div(a,b) << '\n';
    
    return 0;
}

앞서 ops library를 만들 때 생성했던 header file를 include 해서 library 함수들을 이용한다.

main을 컴파일하기 위한 Makefile은 아래와 같다.

Makefile

CC=g++
SRCS=./main.cpp
OBJS=$(notdir $(SRCS:%.cpp=%.o))

TARGET=main

LIBS_FLAG=-lops
LIB_DIRS_FLAG=-L./ops/libs/

build: $(OBJS)
	$(CC) -o $(TARGET) $(OBJS) $(LIBS_FLAG) $(LIB_DIRS_FLAG)

clean:
	rm -rf *.o

$(OBJS): $(SRCS)
	$(CC) -o $@ -c $<

main.cpp를 object 파일로 컴파일 한 뒤 바이너리 파일을 생성할 때 library option을 추가한다.

-lops는 libops라는 library 파일을 연결한다는 의미이고, -L./ops/libs/ 는 library 파일이 위치한 디렉터리를 의미한다.

앞서 ops library를 static, dynamic 두 가지 방식으로 ops/libs 디렉터리에 생성할 수 있도록 했었다.

ops/libs 디렉터리에 libops.a라는 파일이 있으면 static library로 링크되고, libops.so라는 파일이 있으면 dynamic library로 링크된다.

static library link

ops디렉터리에서 make static을 하면 libs/libops.a가 생성된다. 그리고 프로젝트 루트에서 make build를 하면 static library로 링크된 main 프로그램이 생성된다.

dynamic library link

ops디렉터리에서 make dynamic을 하면 libs/libops.so가 생성된다. 그리고 프로젝트 루트에서 make build를 하면 dynamic library로 링크된 main 프로그램이 생성된다.

아무것도 하지 않고 실행하면 결과는 아래와 같다.

dyld[97547]: Library not loaded: libs/libops.so
  Referenced from: <B3CFBE82-2733-3ADB-A12C-EA2032C62692> /Users/tyler/workspace/temp/cpp_lib/main
  Reason: tried: 'libs/libops.so' (no such file), '/System/Volumes/Preboot/Cryptexes/OSlibs/libops.so' (no such file), 'libs/libops.so' (no such file)
[1]    97547 abort      ./main

dynamic library를 찾을 수 없기 때문이다.

시스템이 참조하게 되는 기본 library path에 libops.so를 생성해주거나, 아니면 export를 통해 LD_LIBRARY_PATH를 지정해주어야 한다.

아래와 같이 실행해주어야 한다.

export DYLD_LIBRARY_PATH=$DYLD_LIBRARY_PATH:/Users/tyler/workspace/temp/cpp_lib/ops/libs
./main

mac os를 사용하는 경우, 위와 같이 DYLD_LIBRARY_PATH로 설정해주어야 한다. linux에서는 LD_LIBRARY_PATH이다.
