첫번째 풀이

#include <iostream>
#include <algorithm>
#include <climits>
#include <vector>
#include <queue>
#include <map>
#include <unordered_map>
#include <set>
#include <unordered_set>
#include <bitset>
#include <cstring>
#include <cmath>
using namespace std;
using ll = long long;
int n;
ll values[100000] = {0,};
ll segt[262144] = {0,};
void segt_init(int node, int l, int r){
    if(l==r){
        segt[node] = l;
        return;
    }
    int mid = (l+r)/2;
    segt_init(node*2, l, mid);
    segt_init(node*2+1, mid+1, r);
    if(values[segt[node*2]] < values[segt[node*2+1]]){
        segt[node] = segt[node*2];
    } else {
        segt[node] = segt[node*2+1];
    }
}
ll segt_calc(int node, int l, int r, int start, int end){
    if(l==start && r==end)
        return segt[node];
    int mid = (l+r)/2;
    if(end <= mid)
        return segt_calc(node*2, l, mid, start, end);
    else if(mid+1 <= start)
        return segt_calc(node*2+1, mid+1, r, start, end);
    else {
        int left_min_idx = segt_calc(node*2, l, mid, start, mid);
        int right_min_idx = segt_calc(node*2+1, mid+1, r, mid+1, end);
        if(values[left_min_idx] < values[right_min_idx])
            return left_min_idx;
        else
            return right_min_idx;
    }
}
ll ans = -1;
void calc(int start, int end){
    if(end < start)
        return;
    int min_height_idx = segt_calc(1, 0, n-1, start, end);
    ans = max(ans, (end-start+1)*values[min_height_idx]);
    calc(start, min_height_idx-1); 
    calc(min_height_idx+1, end); 
}
void solve(){
    ans = -1;
    for(int i = 0; i<n; i++){
        cin >> values[i];
    }
    segt_init(1, 0, n-1);
    calc(0, n-1);
    cout << ans << '\n';
}
int main(){
    ios::sync_with_stdio(0);
    cin.tie(0);
    while(1){
        cin >> n;
        if(n==0)
            break;
        solve(); 
    }
    return 0;
}

이 문제에서는 최솟값이 중요하다. 왜냐하면 특정 구간의 넓이는 해당 구간의 최솟값으로 제한되기 때문이다.

n의 최댓값은 100000이다. 각 구간의 최솟값을 찾을 때마다 for문을 돈다면 시간초과에 걸리게 된다. 따라서 segment tree로 특정 구간의 최솟값을 저장해두어야 한다.

segment tree를 이용해서, 모든 구간을 순회하며 최솟값을 찾는다면, 이는 여전히 O(n^2) 으로 시간초과가 걸리게 된다. 따라서 탑-다운 분할 정복을 사용해서 답을 구해야 한다.

n이 8이라고 한다면 다음의 과정을 거쳐서 답을 구한다.

  1. index 0~7 구간의 최솟값을 segment tree로 구해서 넓이를 구한다.
  2. 최솟값에 해당하는 index를 기준으로 왼쪽과 오른쪽으로 구간을 나누어서 1번의 과정을 반복한다.

최솟값이 여러개라면 그중의 아무거나를 통해 분할하면 된다.

히스토그램에서 가장 큰 넓이를 a~b구간에서 찾는다고 해보자.

  1. 만약 가장 큰 넓이가 a~b구간 전체의 너비로 이루어진 넓이라면, a~b구간의 최솟값을 찾아서 구할 수 있다.
  2. 만약 가장 큰 넓이가 a~b구간이 아닌 하위 구간의 너비로 이루어진 넓이라면, 이는 a~b구간에서 최솟값을 제외해야 가능한 넓이가 된다. a~b구간의 최솟값으로 이루어질 수 있는 최대의 넓이는 1에서 이미 구했기 때문이다.

따라서 위에서 사용한 알고리즘으로 최대의 넓이를 구할 수 있게 된다.

두번째 풀이

#include <iostream>
#include <algorithm>
#include <climits>
#include <vector>
#include <queue>
#include <map>
#include <unordered_map>
#include <set>
#include <unordered_set>
#include <bitset>
#include <cstring>
#include <cmath>
using namespace std;
using ll = long long;
int n;
int values[100001] = {0,};
ll ans = 0;
void solve(){
    cin >> n;
    for(int i = 0; i<n; i++){
        cin >> values[i];
    }
    vector<int> stk;
    stk.push_back(0);
    for(int i = 1; i<=n; i++){
        if(values[i] < values[stk.back()]){
            int last_idx;
            while(stk.size() > 0 && values[stk.back()] > values[i]){
                ans = max(ans, (ll)values[stk.back()]*(i-stk.back()));
                last_idx = stk.back();
                stk.pop_back();
            } 
            values[last_idx] = values[i];
            stk.push_back(last_idx);
        }
        stk.push_back(i);
    }
    cout << ans;
}
int main(){
    ios::sync_with_stdio(0);
    cin.tie(0);
    solve(); 
    return 0;
}

같은 문제인 https://www.acmicpc.net/problem/1725의 입력을 기준으로 풀었다.

stack을 사용한 풀이이다.

주어진 히스토그램을 index 0~n-1 에 저장하고, index n 에는 0을 저장한다. 그리고 index 0부터 n까지 순회한다. 이 과정에서 stack에, 이전 index의 높이보다 현재 index의 높이가 더 큰 경우 push하고 다음으로 넘어간다. 만약 이전 index의 높이가 더 크다면, 현재 index의 높이와 같아질 때까지 stack에서 pop을 하며 넓이를 구해준다. pop을 하면서 넓이를 구해줄 수 있는 이유는, 증가하는 높이에 대해서만 히스토그램에 저장을 했기 때문에 pop된 index의 높이만큼이 이후에도 보장이 되기 때문이다. pop이 모두 끝나면 마지막으로 pop된 히스토그램 막대기의 높이를 현재 index의 높이로 바꾸어주고, 다시 stack에 넣어준다.

느낀점

세그먼트 트리를 이용해서 최솟값을 구해본다는 생각까지는 했는데, 분할 정복으로 구하는 것은 떠올리지 못했다. 더 노력이 필요하다.
