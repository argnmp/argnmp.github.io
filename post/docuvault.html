프로젝트 개요

jwt를 기반으로, 백엔드에서 사용이 가능한 문서 관리 서비스입니다. 자주 사용되는 문서 작성 및 파일 업로드 기능을, 이 서비스를 사용하여 대체하는 것을 목표로 진행한 프로젝트입니다.

사용자는 docuvault 서비스의 dashboard 에 로그인 하여, 문서와 관련된 여러 작업을 수행할 수 있습니다.

사용자의 서버에서는 로그인을 통한 access key jwt를 이용해서 (1)작성한 문서들에 대한 metadata 정보를 받고, (2)특정 문서에 대해 열람 가능한 token을 발급 받을 수 있습니다.

metadata를 이용해서 사용자의 서버에서는 웹 페이지를 렌더링 할 수 있고, 열람 가능한 token을 이용해서는 ssr 또는 csr 방식으로 렌더링이 가능합니다.

외부 서비스를 이용하기 보다는, 필요한 기능을 수행하는 서버들을 직접 작성했습니다.

기술 스택

  • rust
    • backend server들을 tokio runtime과 axum 라이브러리를 이용해서 작성했습니다.
  • db
    • postgresql를 이용해서 문서 데이터를 저장합니다.
    • redis를 이용해서 auth에 필요한 blacklist와 refresh token을 관리합니다. 이미지와 같이 file size가 작은 경우 caching하는 용도로 사용됩니다.
  • docker
    • docker를 이용해서 각 서버를 image로 빌드합니다.
    • docker compose를 이용하여 각 서버들을 container로 실행하고 내부 네트워크로 서로 통신할 수 있도록 했습니다.
  • javascript
    • react.js를 이용해서 간단한 dashboard를 작성했습니다.

구현 사항

설계

로그인

jwt를 이용해서 user를 인증할 수 있습니다.

client는 로그인을 하면 access token과 refresh token을 받고, 이 token을 통해 docuvault의 서비스를 이용할 수 있습니다.

로그아웃 구현에 필요한 blacklist의 저장 목적으로 redis를 사용합니다.

jwt access token refresh를 위한 refresh token의 관리 목적으로 redis를 사용합니다.

문서 관리 기능

dashboard 에서는 로그인 이후 docuvault의 rest api를 이용하여 문서 관리 기능을 사용할 수 있습니다.

  • 문서 작성, 삭제, 수정이 가능합니다.
  • 문서 작성 과정에서 이미지 및 파일 업로드가 가능합니다.
  • 문서에 여러개의 scope를 지정해서, metadata 요청 시 scope에 해당하는 문서들만 가져올 수 있습니다.
  • 문서에 tag를 추가할 수 있고, metadata 요청 시 tag를 이용해서 문서 필터링이 가능합니다.
  • sequence를 설정하여, 여러개의 문서들을 순서가 있는 문서의 묶음으로 만들 수 있습니다. 순서를 조정할 수 있으며, 연결된 문서에 대한 metadata 요청이 가능합니다.

문서 변환 기능

pandoc은 문서 변환 오픈소스 프로그램입니다. convert server에서는 이 프로그램을 이용해서, 마크다운으로 작성된 문서를 여러 포맷으로 변환합니다.

WAS에서 문서 변환 요청을 convert server로 보내고, 비동기로 문서 변환 작업을 실행합니다. 문서 변환 완료 시, file server에 결과물을 저장하고 db에 상태값을 업데이트합니다.

파일 업로드 및 다운로드 기능

file에 관한 요청은 file-reverse-proxy server로 보내지고, 이 서버에서는 hashing을 통해 요청을 docker-compose로 구성된 file server에 분산 시킵니다.

문서 요청 기능

access token을 이용하여 문서에 대한 publish token을 생성할 수 있습니다. 이 token을 통해서, 해당 문서에 대해 열람이 가능합니다. 두 가지 방식의 구현이 가능합니다.

  • publish token을 생성한 서버에서 직접 publish token을 이용해 문서 data를 가져온 뒤, 문서 data를 client에 제공하는 방법
  • publish token을 생성한 서버에서 publish token을 client에 제공하고, client에서 publish token을 이용하여 docuvault 서비스에서 문서 data를 가져온 뒤, client side rendering을 이용하는 방법
