Taehyeon Kim

2024-02-03

문제

https://www.acmicpc.net/problem/1493

풀이

주어진 큐브의 크기가 큰 것부터 박스에 집어 넣는다. 그렇게 되면, 나머지 공간을 세 개의 작은 공간으로 나눌 수 있다. 이 공간에 대해서 똑같이 박스를 채워 넣는다.

큰 것부터 별다른 판단 없이 박스에 넣어도 되는 이유는, 큐브가 모든 변이 동일한 정육면체이기 때문이다. 따라서 빠르게 정육면체에 넣을 수 있다.

메모리 초과 코드

#include <iostream>
#include <algorithm>
#include <climits>
#include <vector>
#include <queue>
#include <map>
#include <unordered_map>
#include <set>
#include <unordered_set>
#include <bitset>
#include <cstring>
#include <cmath>
using namespace std;
using ll = long long;
struct box{
    int x, y, z;
    bool operator<(const box& t) const {
        return x < t.x || y < t.y || z < t.z;
    }
    void print(){
        printf("%d, %d, %d\n", x, y, z);
    }
};
int n;
box org;
box cubes[20];
int cubes_count[20] = {0,};

int ans = 0;
bool calc(box target, int next_cube){
    if(target.x==0 || target.y==0 || target.z==0)
        return true;
    for(int i = next_cube; i>=0; i--){
        if(cubes_count[i]==0)
            continue;
        box cur = cubes[i];
        if(target < cur)
            continue;
        cubes_count[i] -= 1;
        ans += 1;
        bool r1 = calc({cur.x, target.y-cur.y, cur.z}, i);
        bool r2 = calc({target.x-cur.x, target.y, cur.z}, i);
        bool r3 = calc({target.x, target.y, target.z-cur.z}, i);
        if(r1 && r2 && r3)    
            return true;
        else
            return false;
    }
    return false;
}

void solve(){
    cin >> org.x >> org.y >> org.z;
    cin >> n;
    int a, b;
    for(int i = 0; i<n; i++){
        cin >> a >> b;
        int k = pow(2, a); 
        cubes[a] = {k, k, k};
        cubes_count[a] = b;
    }
    if(calc(org, 19)){
        cout << ans;
    }
    else {
        cout << "-1";
    }
}
int main(){
    ios::sync_with_stdio(0);
    cin.tie(0);
    solve(); 
    return 0;
}

이상한 것은 1번과 3번의 경우, struct를 함수의 인자로 사용하는 것과 함수의 본문에서 사용하는 것에서 차이가 발생하는데 메모리 사용량이 큰 폭이서 차이가난다.

또한 2번과 3번의 경우, 3번에서는 struct로 한번 변환을 한 뒤에 사용을 하는데, 변환하지 않는 것보다 더 적은 시간이 소요된다.

느낀점

나는 이 문제를 메모리 초과 때문에 많이 고생을 했다. 처음에 풀 때는, 다음과 같은 방법으로 해보았다.

  • 가장 큰 큐브부터 박스에 넣고, 남은 부분을 네 조각으로 나눈다
  • 박스의 가장 짧은 길이가 길수록 우선순위가 높은 priority_queue에 네 조각을 넣는다.
  • priority_queue에서 박스를 하나 빼서 위의 과정을 반복한다.

하지만, 이 방법은 메모리 초과가 발생하고 느렸다. 왜냐하면 일단 priority_queue를 사용할 필요가 없었다. 어차피 조각을 나누고 하는 과정에서 무조건 큰 큐브부터 반복문을 돌면서 맞추었기 때문이다. 두번째로 priority_queue에서 많은 조각들이 들어가다보니 메모리 초과가 발생했다. 그래서 큐브들을 모아서 더 큰 정육면체를 만들어서 넣는 방법도 사용했지만 해결되지 않았다.

그래서 다른 사람의 풀이를 보고 재귀 호출로 코드를 작성했다.. 하지만 메모리 초과가 발생했다. 시간이 오래 걸려서 메모리 초과의 이유를 알아냈고, 발생한 이유 중 가장 큰 지분을 차지한 것이 struct이다. struct를 재귀 함수 호출시 함수의 인자로 사용하는데, 함수 호출문에 struct를 넣더라도(calc({a-b, b-c, c-a})와 같이) 계산 결과를 stack에 한번 복사를 한뒤 register에 넣어서 호출을 하는 방식이 사용되기 때문이다. struct를 함수 인자로 사용 시 스택 사용량 이 문제에서는 특히 재귀 호출이 많이 발생했기 때문에, 메모리 초과 문제가 발생하였다. 앞으로는 재귀 호출 코드를 작성할 때 이러한 점을 주의해야겠다.
