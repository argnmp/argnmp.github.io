풀이

#include <iostream>
#include <algorithm>
#include <climits>
#include <vector>
#include <queue>
#include <map>
#include <unordered_map>
#include <set>
#include <unordered_set>
#include <bitset>
#include <cstring>
#include <cmath>
using namespace std;
using ll = long long;
int n, k;
ll mod = 1000000007;
void solve(){
    cin >> n >> k;
    ll a =  1;
    for(int i = n; i>=n-k+1; i--){
        a = (a*i)%mod;
    }
    ll b = 1;
    for(int i = k; i>=1; i--){
        b = (b*i)%mod;
    }
    ll remainder = 1;
    ll t = mod-2;
    while(t>0){
        if(t%2==1) 
            remainder = (remainder * b) % mod;
        b = (b*b) % mod;
        t/=2;
    }
    cout << (a * remainder) % mod;
}
int main(){
    ios::sync_with_stdio(0);
    cin.tie(0);
    solve(); 
    return 0;
}

먼저 이항계수의 성질을 사용하는 방법을 생각해볼 수 있다. nCk = n-1Ck-1 + n-1Ck 이다. 이를 이용해서, 재귀적으로 연산하며 nCk를 저장한 뒤 재사용하는 방법이 있다. 하지만 n가 k가 최대 4000000 으로, 이 방법은 사용하기 어렵다.

이 문제에서는 페르마의 소정리를 이용해야 한다.

일단 이항계수는 다음과 같이 계산한다.

nCk =  n! / k! * (n-k)! = (n-k+1)! / k! = (n-k+1)! * (k!)^-1

n가 k에 값을 넣어서 위의 식을 먼저 계산하는 것이 불가능하다. 왜냐하면 최대 4000000 으로, 자료형의 범위를 벗어날 수 있기 때문이다. 따라서 곱셈을 해주는 과정에서 나머지 연산을 계속해주어야 하는데, 분수는 나머지 연산을 해주기 어렵기 때문에 여기서 페르마의 소정리를 사용한다.

페르마의 소정리는 다음과 같다.

a^(p-1) ≡ 1 (mod p)
a*a^(p-2) ≡ 1 (mod p)
a^(p-2) ≡ a^-1 (mod p)

이를 위 식에 적용하면 (p는 문제에서 주어진 값, 1000000007이다.) 아래와 같다.

((n-k+1)! * (k!)^-1) % p
= (((n-k+1)! % p) * ((k!)^-1 % p)) % p
= (((n-k+1)! % p) * (((k!)^(p-2) % p) % p)) % p

이제 곱셈만 남았으므로, 매 곱셈마다 p로 나머지 연산을 해주면 된다.

주의할 점은 (k!)^(p-2)에서 그냥 곱셈만 한다면 굉장히 시간이 오래 걸린다. 따라서 boj 12850 본대 산책2 에서 사용했던 O(lgN) 의 거듭제곱 방식을 사용해야 한다.

느낀점

페르마의 소정리를 알게 되었다.
