update vs updateMany

prisma 에서 update를 하는 구문은 다음과 같다

const update = await prisma.user.update({
  where: {
    id: 6,
  },
  data: {
    posts: {
      deleteMany: [{ id: 7 }],
    },
  },
})

하지만 update의 where 문에서 여러개의 column 을 조건으로 거는 경우에는 update를 사용할 수 없다. 문제는 multiple column 이 unique 하더라도 update를 사용할수 없다는 것이다. 따라서 updateMany를 사용해야 한다.

그러나 이 updateMany도 문제가 있다. 바로 updateMany 를 사용하는 경우 relation update query가 불가능 하다는 것이다.

다음의 예시를 살펴보자.

relation example

model User {
  id                    Int                @id @default(autoincrement())
  provider              String
  provider_id           String?
  email                 String             @db.VarChar(50)
  ...
  UserProfile           UserProfile?
  ...
  @@unique([provider, provider_id])
  @@unique([provider, email])
}

model UserProfile {
  id         Int      @id @default(autoincrement())
  user_id    Int      @unique
  ...
  User       User     @relation(fields: [user_id], references: [id], onDelete: Cascade)
}

db schema는 위와 같은 구조로 설정되어 있다. one to one relation 이다보니 UserProfile 쪽에서 FK를 걸어놓은 방식이다.

처음에는 User 을 통해서 UserProfile를 update 하려고 했으나, User을 column 을 검색하는 데는 multiple column이 필요했고, 이는 updateMany를 사용해야했으며, updateMany를 사용하면 UserProfile 에 대한 relation update가 불가능해진다.

결국 UserProfile을 user_id를 사용해서 업데이트하는 방법을 택했고, 이 경우에는 하나의 column 즉 user_id 만 필요하기 때문에 update를 사용할 수 있게 되어 User의 column도 수정이 가능해진다.
