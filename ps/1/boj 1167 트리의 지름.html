문제

코드

#include <iostream>
#include <algorithm>
#include <climits>
#include <vector>
#include <queue>
#include <map>
#include <unordered_map>
#include <set>
#include <unordered_set>
#include <bitset>
#include <cstring>
using namespace std;
using ll = long long;
int n;
struct edge{
    int u, v, w;
};
vector<edge> graph[100001];

struct ret{
    int v, dist;
};
int visited[100001] = {0,};
ret dfs(int v){
    visited[v] = 1;   
    ret return_value = {v, 0};
    for(edge next: graph[v]){
        if(visited[next.v] == 0){
            ret res = dfs(next.v);
            if(res.dist + next.w >= return_value.dist){
                return_value = {res.v, res.dist + next.w};
            }
        }
    }
    return return_value;
}
void solve(){
    cin >> n;
    int u, v, w;
    for(int i = 1; i<=n; i++){
        cin >> u; 
        while(1){
            cin >> v;
            if(v!=-1){
                cin >> w;
                graph[u].push_back({u,v,w});
            } 
            else {
                break;
            }
        }
    }
    ret end = dfs(1);
    memset(visited, 0, sizeof(visited));
    ret ans = dfs(end.v);
    cout << ans.dist << '\n';
}
int main(){
    ios::sync_with_stdio(0);
    cin.tie(0);
    solve(); 
    return 0;
}

풀이

vertex 하나(x)를 무작위로 골라서 그곳으로 부터 가장 먼 vertex(y)를 찾는다. 그리고 y에서 가장 먼 vertex 까지의 거리가 트리의 지름이 된다.

이 알고리즘에 대한 증명은 아래의 사이트에 있다. https://blog.myungwoo.kr/112

위 사이트의 설명의 3번째 경우의 수에 대해 조금 덧붙여 보겠다. 사이트의 설명과 같이 d(u,v) 를 vertex u와 vertex v 사이의 거리로 정의한다.

하나의 vertex를 공유하는 경우

x에서 가장 먼 vertex가 y이므로, d(t,y) >= max(d(t,u), d(t,v)) 이다. d(t,u)보다 d(t,v)가 더 길다고 해보자. d(t,y)와 d(t,v) 가 같지 않다면, d(t,y)가 더 커야 하는데, 이렇게 될 경우 d(u,v)가 트리의 지름이 될 수 없다. 따라서, d(t,y)와 d(t,v)가 같지 않는 한 모순된다. 만약 같다면, y에서 가장 먼 vertex를 구했을 때 v에서 구한 것 과 마찬가지로 u가 나오기 때문에 트리의 지름을 구할 수 있다.

vertex를 공유하지 않는 경우

위쪽의 점을 t1, 아래쪽의 점을 t2라고 해보자. x에서 가장 먼 vertex가 y이므로, d(t1, y) >= d(t1,t2) + max(d(t2, u), d(t2, v)) 이다. d(t2,u)보다 d(t2, v)가 더 길다고 해보자. d(t1, y)와 d(t1, t2) + d(t2, v)가 같지 않다면, d(t1, y)가 더 커야 하는데, 이 경우, d(u,v)가 트리의 지름이 될 수 없다. 따라서 모순이 된다.

d(t1, y)와 d(t1,t2) + d(t2, v)가 만약 같다면, y에서 가장 먼 vertex를 구했을 때의 길이가 d(u,v) 보다 길어지기 때문에 모순이 된다.

결론적으로 위의 사이트에서 소개된 알고리즘의 방식으로 트리의 지름을 구할 수 있다.

느낀점
