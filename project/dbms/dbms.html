개요

  • 정형수 교수님의 ITE2038 ‘데이터베이스 시스템 및 용용’ 강의에서 진행한, dbms 소프트웨어 제작 프로젝트입니다.
  • dbms에서 필수적인 기능들을 layered architecture로 구현하였고, 데이터베이스의 기초적인 이론을 프로젝트에 적용하였습니다.
  • 한 학기 동안 진행한 프로젝트 입니다.

프로젝트 링크

https://github.com/kimtahen/ite2038

프로젝트 기간

2022.09.23 ~ 2022.12.20

기술 스택

  • c++ make

구현 사항

  • disk space manager
    • ite2038/project2 at master · kimtahen/ite2038
    • database page 단위로, disk에서 I/O를 하는 disk space manager api를 작성하였습니다.
  • index manager
    • ite2038/project2 at master · kimtahen/ite2038
    • ite2038/project2 at master · kimtahen/ite2038
    • 앞서 제작한 disk space management layer의 위에, key-value의 insert와 delete가 가능한 on-disk b+ tree index api를 작성하였습니다.
  • buffer manager
    • ite2038/project3 at master · kimtahen/ite2038
    • disk I/O의 느린 속도를 향상시키기 위해서, LRU policy를 사용하는 buffer layer api를 작성하여 성능을 큰 폭으로 개선하였습니다.
  • lock table
    • ite2038/project4 at master · kimtahen/ite2038
    • transaction layer를 추가하기에 앞서서, lock table을 구현하였습니다. 여러 thread가 record에 동시에 접근할 때, 동기화를 해주는 역할을 합니다.
  • lock manager, transaction manager
    • ite2038/project5 at master · kimtahen/ite2038
    • S2PL 기법을 적용한 concurrency control을 추가하여, 여러 thread에서 동시에 transaction을 진행할 수 있도록 lock manager와 transaction manager를 작성하였습니다.
  • log & recovery
    • ite2038/project6 at master · kimtahen/ite2038
    • log와 recovery api를 추가하여, dbms가 실행중 강제로 종료되었을 때, 복구가 가능하도록 기능을 추가하였습니다.
    • 이 기능에 대해서는 완벽히 구현하지 못하였습니다.
