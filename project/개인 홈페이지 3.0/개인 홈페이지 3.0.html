개요

개인 블로그로 사용하기 위해 제작한 3번째 블로그입니다. 그동안 두번의 블로그 제작을 거쳤습니다. 하지만 복잡성이 매우 높았고, 단순명료함과 가독성에 초점을 맞춘 블로그 제작을 목표로 하였습니다.

위의 목표를 달성하기 위해서는 가장 자원을 효율적으로 사용할 수 있는 방법을 택해야 했고, 결론적으로 serverless 방식을 택했습니다. serverless 방식을 사용함으로써 여러 무료로 제공되는 정적 파일 호스팅 서비스를 사용할 수 있게 되었고, 그중의 하나인 github pages를 블로그 서비스에 사용하기로 결정하였습니다.

정적 블로그를 생성하여 배포하기 위해서는 로컬에서 모든 렌더링 작업을 수행해야합니다. 마크다운으로 작성한 파일들을 빠르고 안정적으로 렌더링 하기 위해서, rust와 python을 사용하였습니다.

구현 사항

마크다운 파싱 및 렌더링

먼저 obsidian이라는 마크다운 편집기로 생성된 로컬의 마크다운 및 바이너리 파일과 디렉터리를 파싱합니다. 마크다운 파일 내에 작성된 property 들을 가져옵니다. 이 리소스들은 하위 디렉터리와 파일들로 구성되어 있으며, 하나의 루트 디렉터리로 부터 트리 구조를 형성합니다. 리소스의 집합은 여러개의 루트 디렉터리로 구성될 수 있으며, 파싱을 하는 과정에서 이 디렉토리들을 하나의 루트 디렉터리를 가진 트리 구조로 재편성합니다. 또한 결과물이 업로드 될 base url을 기준으로 상대 경로를 생성합니다.

이후 트리구조 내의 모든 리소스들에 대해 여러개의 스레드를 사용하여 동시에 렌더링을 합니다.

  • 디렉터리 리소스의 경우, 먼저 결과물이 위치하게 될 최상위 디렉터리 하위에 새로운 디렉터리를 생성합니다. 이후 템플릿 엔진을 사용하여 하위 리소스들에 대한 하이퍼링크를 가진 index html 파일을 생성합니다.
  • 바이너리 파일의 경우 앞서 생성한 상대경로를 따라 파일을 복사합니다.
  • 마크다운 파일의 경우 먼저 html로 변환을 진행합니다. 이 과정에서 다른 리소스에 대한 의존성이 있는경우, 앞서 생성한 상대경로를 참조하여 새롭게 하이퍼링크를 생성합니다. 템플릿 엔진을 사용해서, 마크다운 에서 변환된 html 파일을 포함하는 문서 html을 생성합니다.

검색 인덱스 생성 및 검색 기능 구현

serverless의 정적 블로그에서 검색기능을 구현하기 위한 방법으로, 미리 인덱스 파일을 생성하고 wasm을 통해 import 하여, 검색어를 포함하는 문서를 찾는 방식을 사용했습니다. 한국어의 경우 영어와 달리 조사가 명사와 결합하여 하나의 단어를 이루기 때문에, 단순히 띄어쓰기 단위로 단어를 구별한 뒤 의미있는 단어만 골라내는 방식을 사용할 수 없습니다. 대부분의 한국어 nlp 라이브러리가 python으로 작성되어 있어서, python을 sub process로 실행하여 사용하였습니다.

python script에서는 loop를 돌며 stdin 으로 문장을 입력받아서 명사를 추출한 뒤 json format으로 결과물을 stdout 으로 출력합니다. rust로 작성된 메인 프로그램에서는 먼저 지정된 개수의 sub process로 python script를 실행하여, process pool을 구성합니다. 위의 파싱 과정에서 각 스레드에서는 process pool의 process에 stdin으로 문장을 입력하고, stdout으로 나오는 결과물에 대해서 xor filter를 사용한 인덱스를 생성합니다. 그리고 이 인덱스들을 모두 모아서 serialize 과정을 거쳐, 결과물이 위치할 디렉터리 하위에 바이너리 파일로 저장합니다.

검색 인덱스의 import는 rust로 작성된 WASM을 사용하여 이루어집니다. client에서 실행되는 wasm에서는 바이너리 파일을 fetch 하여 deserialize하여 xor filter 인덱스를 생성합니다. 검색어에 대해 띄어쓰기를 기준으로 나누어서, 모든 문서에 대해서 각 검색어가 포함되는지 여부를 검증하고 priority를 부여합니다. priority 순서대로 검색어에 해당하는 문서를 정렬하여 보여줍니다.

client side rendering

serverless의 목적에 맞게 정적 웹 사이트 호스팅이 가능하도록 렌더링을 하지만, github page와 같은 서비스는 무료인 대신 느리다는 단점이 있습니다. 이를 해결하기 위해서 블로그에 client side rendering 기능을 추가했습니다. url을 통해 정적 자원에 접근을 하면, client에서는 html, js, wasm을 다운로드하여 렌더링을 합니다. 이후 렌더링이 완료되고, 렌더링 된 페이지의 미리 fetch 할 대상 으로 선정되어 있는 url 들에 대해서 백그라운드로 fetch를 진행한 뒤 저장을 합니다. client에서 미리 로드된 페이지에 대하여 링크를 클릭하면 정적 자원에 접근을 하는 대신 dom element를 교체하여 다시 렌더링합니다. 이러한 방식을 사용해서 빠르게 로딩을 할 수 있습니다. 로컬에서 렌더링하는 과정은 SSR이라고 볼 수 있고, client에서 백그라운드로 fetch 후 dom element를 교체하는 과정은 CSR이라고 볼 수 있습니다. 이 두 방식을 하이브리드하여 접근성을 극대화 하였습니다.
