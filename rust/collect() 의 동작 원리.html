collect() 의 동작 원리

alias undefined
author Taehyeon Kim
created at 2023-07-27
tags

코드를 살펴보자

fn main() {
    let num_vec = vec![2, 4, 6];
    let new_vec = num_vec.iter().map(|x| x * 100).collect::<Vec<i32>>();
    println!("{new_vec:?}");
}

[200, 400, 600]

iter()과 같은 함수로 iterator을 생성하면 map메서드를 호출이 가능하고, 이후 collect메서드를 사용해서 다시 vector로 산출해 낼 수 있다. 이 과정에서는 map메서드를 호출하는 순간 100을 곱하는 연산이 수행되는 것이 아닌, collect 메서드를 호출해야 연산이 수행되고 값이 산출된다. 어떤 방식으로 구현이 되는지 궁금해서 소스코드를 살펴보게 되었다.

iter()

iter메서드를 호출하기 위해서는 Vec이 iter메서드를 가지고 있어야 한다. 하지만 Vec의 정의에 따르면 직접 iter을 구현해 놓지 않았고, Vec이 implement 한 Deref 를 통해서 iter 메서드를 사용할 수 있다.

https://kimtahen.com/post/Deref-trait#hidauto-deref

이 글에서 auto deref 를 통해서 호출이 가능한 이유를 알 수 있다.

Vec 은 Deref를 다음과 같이 implement 한다.

#[stable(feature = "rust1", since = "1.0.0")]
impl<T, A: Allocator> ops::Deref for Vec<T, A> {
    type Target = [T];

    fn deref(&self) -> &[T] {
        unsafe { slice::from_raw_parts(self.as_ptr(), self.len) }
    }
}

위의 코드에서 num_vec.iter()는 (*num_vec).iter()로 변환이 되고 다시 (*(&[i32])).iter()로 변환이 되어 실행된다.(Vec이 i32인 요소들로 구성되어 있기 때문에) 그러면 결국 [i32].iter()의 호출과 같고 여기서 [i32]는 slice이다. 따라서 slice가 iter 메서드를 가지고 있는 지 확인해야 한다.

slice는 primitive type이고, 예상한대로 iter 메서드를 가지고 있다. 다음과 같이 구현되어 있다.

#[stable(feature = "rust1", since = "1.0.0")]
#[inline]
pub fn iter(&self) -> Iter<'_, T> {
    Iter::new(self)
}

현재 self 는 [i32]이고, Iter::new(self)(Iter::new(&[i32])) 를 반환한다.

그럼 이제 rust std에 Iter 을 찾아보자 그러면 다음과 같이 Iter struct 가 한 개가 아니라 수없이 많다.

이 중에 과연 뭘까? slice의 iter 메서드를 구현해 놓은 라이브러리를 보면 된다.

https://doc.rust-lang.org/src/core/slice/mod.rs.html#732

위 링크의 732번째 줄에 구현이 되어있는 것을 확인할 수 있다.

같은 파일의 34번째 줄과 43번째 줄을 보면

https://doc.rust-lang.org/src/core/slice/mod.rs.html#34 https://doc.rust-lang.org/src/core/slice/mod.rs.html#43

mod iter;

...

pub use iter::{Iter, IterMut};

이러한 코드가 작성되있는 것을 확인할 수 있다. mod iter 은 같은 디렉터리에 위치하는 iter 이라는 파일을 모듈로 정의한다는 의미이고, pub use iter::{Iter, IterMut}; 은 그렇게 정의한 iter 파일로 부터 Iter과 IterMut struct를 사용한다는 의미이다.

이러한 구조로 되어있다. slice 의 iter 메서드는 mod.rs파일에 정의되어 있고, 따라서 사용한 Iter struct는 같은 디렉터리의 iter.rs에 정의되어 있음을 알 수 있다. 그렇기 때문에 slice의 iter메서드에서 사용한 Iter struct는 slice::iter 이라고 할 수 있다.

따라서 rust std의 std::slice::Iter을 확인하면 된다. https://doc.rust-lang.org/std/slice/struct.Iter.html

map()

Iter struct는 Iterator을 implement 하고 있고 이 덕분에 map메서드를 사용할 수 있다. 이는 다음과 같이 구현된다.

iterator! {struct Iter -> *const T, &'a T, const, {/* no mut */}, {
    fn is_sorted_by<F>(self, mut compare: F) -> bool
    where
        Self: Sized,
        F: FnMut(&Self::Item, &Self::Item) -> Option<Ordering>,
    {
        self.as_slice().windows(2).all(|w| {
            compare(&&w[0], &&w[1]).map(|o| o != Ordering::Greater).unwrap_or(false)
        })
    }
}}

macro를 사용해서 Iterator을 implement 하는 것 같은데, 아직 내 실력이 부족해서 이해를 하지 못하겠다.

Iterator trait 에서 map 메서드는 다음과 같이 구현되어 있다.

#[inline]
#[stable(feature = "rust1", since = "1.0.0")]
fn map<B, F>(self, f: F) -> Map<Self, F>
where
    Self: Sized,
    F: FnMut(Self::Item) -> B,
{
    Map::new(self, f)
}

클로저를 인수로 받아서, Map struct 를 반환한다. 그리고 Map::new 는 다음과 같이 구현되어 있다.

https://doc.rust-lang.org/std/iter/struct.Map.html

위 링크는 rust std의 Map 에 대한 정보인데, new 메서드가 나와있지 않다. 그리고 다음과 같이 설명되어 있다.

An iterator that maps the values of iter with f. This struct is created by the map method on Iterator. See its documentation for more.

Iterator의 map 메서드로만 생성되는 struct라고 되어있다. 일부러 호출할 일이 없기 때문에 아마도 new의 설명이 생략되어 있는게 아닌가 싶다.

source를 눌러 구현부를 살펴보면 new메서드는 다음과 같이 구현되어 있다.

#[must_use = "iterators are lazy and do nothing unless consumed"]
#[stable(feature = "rust1", since = "1.0.0")]
#[derive(Clone)]
pub struct Map<I, F> {
    // Used for `SplitWhitespace` and `SplitAsciiWhitespace` `as_str` methods
    pub(crate) iter: I,
    f: F,
}

impl<I, F> Map<I, F> {
    pub(in crate::iter) fn new(iter: I, f: F) -> Map<I, F> {
        Map { iter, f }
    }
}

가장 윗부분에는 우리가 잘 알고 있는 사실이 쓰여져 있다. "iterator은 게으르고 사용되기 전까지 아무것도 하지 않는다"

new메서드에서는 인수로 받은 Iter struct와(위의 코드에서 self를 넘겨주는데, 함수는 Iterator trait에 의해 정의된 것이고 원래는 Iter struct 였다.) 클로저를 Map struct에 넣어서 반환한다.

이를 통해 num_vec.iter().map(|x| x * 100) 까지 실행해도 map 으로 넘긴 클로저가 실제 값으로 실행되지 않았음을 확인할 수 있다.

collect()

Map struct 는 Iterator trait 을 implement 하고 있고, 이 때문에 collect 메서드를 사용할 수 있다.

#[inline]
#[stable(feature = "rust1", since = "1.0.0")]
#[must_use = "if you really need to exhaust the iterator, consider `.for_each(drop)` instead"]
fn collect<B: FromIterator<Self::Item>>(self) -> B
where
    Self: Sized,
{
    FromIterator::from_iter(self)
}

collect 메서드는 turbo fish syntax 를 이용해서 .collect::<Vec<i32>>() 이렇게 호출한다. collect의 정의의 B 의 타입에 Vec<i32> 를 정의하는 것이다. collect의 정의에서 B type은 FromIterator trait을 implement 해야한다는 것을 알 수 있다.

그렇다면 여기서 모든 것의 대상이되는 Self는 어떻게 알 수 있을까? 바로 FromIterator::from_iter(self) 를 보아야 한다. 아래는 FromIterator의 구현부이다.

#[rustc_diagnostic_item = "FromIterator"]
pub trait FromIterator<A>: Sized {
    #[stable(feature = "rust1", since = "1.0.0")]
    fn from_iter<T: IntoIterator<Item = A>>(iter: T) -> Self;
}

FromIterator trait의 from_iter 메서드는 Self를 반환한다. collect 구현 코드를 보면 from_iter(self)의 반환값이 바로 반환이 되고 이를 B라고 정의한다. 따라서 B는 FromIterator::from_iter 측면에서 Self가 되고, 지금 살펴보는 코드에서는 Vec<i32> 가 된다.

그렇기 때문에 collect 매서드에서의 FromIterator::from_iter 은 Vec 에서 implement 한 FromIterator을 의미하게 된다.

결론적으로 collect 메서드는 Vec의 from_iter메서드를 호출하며 Vec<i32>를 반환하는 것을 알 수 있다. Map 객체를 어떻게 Vec으로 만드는 지는 다음에 더 살펴보아야 할 것 같다.
