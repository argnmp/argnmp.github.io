조건부 trait bound

alias undefined
author Taehyeon Kim
created at 2022-07-26
tags

개요

A라는 trait이 있다고 하자. A trait을 implement 해야 B trait도 implement가 가능하도록 만들고 싶은 경우에 대해 살펴본다.

첫번째 경우

trait A {}
trait B : A{}

개요의 상황이다. 위의 코드와 같이 작성하면 된다.

두번째 경우

Debug,Display trait을 implement 할 때 Prints trait을 implement 할 수 있도록 코드를 작성하려면 아래와 같이 하면 된다.

trait Prints: Debug + Display{
    fn debug_print(&self){
        println!("I am: {self:?}");
    }
    fn display_print(&self){
        println!("I am: {self}");
    }
}

세번째 경우

앞서 작성한 Prints trait을 아무나 implement가 가능하게 하는 대신, 각 함수를 trait bound 된 대상이 implement 하고 있는 trait에 따라 사용할 수 있도록 하기 위핸 코드는 아래와 같다.

trait Prints{
    fn debug_print(&self) where Self: Debug{
        println!("I am: {self:?}");
    }
    fn display_print(&self) where Self: Display{
        println!("I am: {self}");
    }
}

Prints를 implement하는 struct가 있다고 할때, Debug를 implement 하는 경우 debug_print의 사용이 가능하고 Display를 implement 하는 경우 display_print의 사용이 가능하다.
