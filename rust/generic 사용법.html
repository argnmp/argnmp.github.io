trait bound

특정한 trait을 가지고 있는 struct을 인수로 받고 싶을 때는 generic을 다음과 같이 사용한다.

generic을 사용하는 방법

trait Greeting {
    fn default(&self)->String{
        String::from("hello world")
    }
    fn special(&self)->String;
}
fn greet<T: Greeting> (item: &T){
    println!("{}",item.default());
    println!("{}",item.special()); 
}

추가: &impl을 사용하는 방법

generic 대신 다음과 같이 사용이 가능하다.

trait Greeting {
    fn default(&self)->String{
        String::from("hello world")
    }
    fn special(&self)->String;
}
fn greet (item: &impl Greeting){
    println!("{}",item.default());
    println!("{}",item.special()); 
}

multiple trait bound

generic을 사용하는 방법

+ syntax

    pub fn ret<T: Display + Clone, U: Clone + Debug> (t: T, u: U) -> (T, U){
        (t, u)
    }

where syntax

    pub fn ret<T, U> (t: T, u: U) -> (T, U) where T: Display + Clone, U: Clone + Debug{
        (t,u)
    }

추가: &impl을 사용하는 방법

+ syntax

    pub fn ret2(t: impl Display + Clone, u: impl Clone + Debug) -> (impl Display + Clone, impl Clone + Debug){
        (t,u)
    }

참조자를 사용하고 싶은 경우 &(impl Display + Clone) 이렇게 감싸주면 된다.

이 코드에서는 Display, Clone trait을 가진 t와 Clone, Debug trait을 가진 u를 받아서 튜플로 반환을 한다. 하지만 impl로 반환을 하면 정의한 trait 만 가진 채로 반환하게 되어서 컴파일 에러가 발생할 수도 있다.

앞서 언급한 generic을 사용한 방법과 impl을 사용한 방법을 비교해보면 된다.
