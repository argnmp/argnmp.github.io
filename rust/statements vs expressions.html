The Rust Programming Language 의 Statements and Expressions 부분을 참고하였다.

Statements VS Expressions

statement와 expression의 차이점

Statements are instructions that perform some action and do not return a value.

Expressions evaluate to a resulting value.

rust에서 statement와 expression의 차이점은 값의 반환 유무이고, 이는 세미콜론으로 구분할 수도 있다. expression은 구문의 끝에 세미콜론을 붙이지 않지만, statement는 끝에 세미콜론을 붙인다. 또한 expression 구문의 끝에 세미콜론을 붙임으로써 statement로 변환할 수 있다.

C언어에서는 아래와 같은 코드가 잘 작동한다.

int main(){
    int x;
    int y;
    x = y = 7;
    printf("%d, %d",x,y);
    return 0;
}

출력 결과

7, 7

이와 달리 rust에서는 c언어에서 사용했던 assign 구문이 expression이 아니기 때문에 값을 반환하지 않아서 아래와 같이 제대로 값이 할당 되지 않는다.

fn main() {
    let mut x;
    let mut y;
    x = y = 6;
    println!("{}, {}", x, y);
}

출력 결과

error[E0277]: `()` doesn't implement `std::fmt::Display`

  |
5 |     println!("{}, {}", x, y);
  |                        ^ `()` cannot be formatted with the default formatter

이처럼 컴파일 에러가 발생한다. 즉 assign은 statement라고 할 수 있다.

expression examples

let x = 6;

expression은 statement의 일부가 될 수 있다. 위의 코드에서 6 은 value 6을 반환하는 expression 이고, let x = 6; 이라는 statement의 일부이다.

함수와 매크로의 호출 또한 expression에 속한다.

공식 문서의 말을 빌리자면 A new scope block created with curly brackets 도 expression 에 속한다.

fn main() {
    let y = {
        let x = 3;
        x + 1
    };

    println!("The value of y is: {}", y);
}

중괄호로 시작해서 중괄호로 끝나는 스코프 블럭 또한 expression 이므로 위와 같은 코드가 가능하다. 위에서 x+1 이 expression이기 때문에 4라는 값을 반환하고, 다시 반환되어 결과적으로 y에 할당된다.

함수의 반환 값

rust에서 함수의 반환 값은 명시적으로 return 구문을 통해서 나타내거나, 마지막 expression의 반환 값 으로 나타낸다.

fn five() -> i32 {
    5
}

위의 코드는 옳다. 5 자체가 expression 이므로 이 함수는 5를 반환 한다.

하지만 다음의 코드는 컴파일 에러가 발생한다.

fn five() -> i32 {
    5;
}

error[E0308]: mismatched types
 --> src\main.rs:4:12
  |
4 | fn five()->i32{
  |    ----    ^^^ expected `i32`, found `()`
  |    |
  |    implicitly returns `()` as its body has no tail or `return` expression
5 |     5;
  |      - help: consider removing this semicolon

5에 세미콜론을 붙이면 statement가 되어 더이상 값을 반환하지 않는다. 함수의 선언 부에서 -> 키워드를 통해 명시적으로 i32 타입의 값을 반환한다고 했는데, 값을 반환하지 않으니 컴파일 에러가 발생한다.

추가

if 문 또한 expression 으로 사용이 가능하다. 따라서 아래와 같은 코드가 가능하다

fn main() {
    let condition = true;
    let number = if condition { 5 } else { 6 };

    println!("The value of number is: {}", number);
}

여기서 if 와 else arms 의 반환 타입이 다른 경우 컴파일 에러가 발생한다.
