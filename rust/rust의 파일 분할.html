binary package

rust에서 main.rs을 진입점으로 실행되는 package를 binary package 라고 한다. binary package에서는 src/main.rs 가 존재한다.

library package

실행하는 package가 아닌, 다른 package에서 사용하기 위한 package를 library package 라고 한다. 보통 다른 패키지에서 이 library package를 use keyword를 사용해 불러온다.

src/lib.rs를 필두로 mod 키워드를 사용해 작성된다.

그렇다면 이 binary package와 library package가 같이 존재하는 package는 어떤 구조일까? 다음과 같다.

binary + library package

이 경우, 프로젝트의 구조는 src 디렉터리 하위에 main.rs와 lib.rs가 공존하는 구조로 되어있다.

lib.rs에서는 mod 키워드를 통해서 module들을 작성하면 되고, main.rs에서는 binary 프로그램을 작성하면 된다.

아래의 예시를 보자.

Cargo.toml은 다음과 같다.

[package]
name = "ppp"
version = "0.1.0"
edition = "2021"

# See more keys and their definitions at https://doc.rust-lang.org/cargo/reference/manifest.html

[dependencies]

package 이름은 ppp이다.

lib.rs에 module은 다음과 같이 정의하였다.

pub mod uppermod{
    pub fn upperfn(){
        println!("print from upper fn");
    }
    pub mod lowermod{
        pub fn lowerfn(){
            println!("print from lower fn");
        }
    }
}

main.rs에서 upperfn과 lowerfn을 사용하는 방법은 다음과 같다.

main.rs

use ppp::uppermod::{self, lowermod};

fn main(){
    uppermod::upperfn(); 
    lowermod::lowerfn();
}

use keyword로 module을 불러오면 된다. 단, use keyword로 불러올 때 최상단에 crate가 아닌 package name을 써주어야 한다.
